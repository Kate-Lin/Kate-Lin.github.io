# Linux

# Linux 常用命令

  • grep : 根据文件的内容进行查找，用于查找文件里符合条件的字符串
    • 直接搜索某文件中的字符串：例如 grep test1 test* —— 查找文件名前缀为 test ，包含 test1 字符串的文件
    • 查找特定时间的目录: grep
    • 递归查找： -r
    • 反向查找： -v
    • 显示行号： -n
  • find ：根据文件的属性进行查找（如文件名、文件大小、所有者、修改时间等）
    • find [path]
  • awk ：对某个文件的特定行执行脚本命令。
    • awk '{print $1}' data2.txt
    • echo "My name is Rich" | awk '{$4="Christine"; print $0}' (输出 My name is Christine )
    • BEGIN 和 END 标识
  • tee : 将输入复制到某文件中
  • sed ：用 script 指令进行增删改（和 awk 不太一样，用的不是 bash）
    • sed -e 4a\newline testfile
    • sed -e 2,5d testfile
    • sed -e 3,$d testfile
    • sed -e 4i\newline testfile

# 管道

例如 ls -l|grep ^-|wc -l

  • ls -l : 列出当前目录下的所有文件
  • grep ^- : 查找行首以 - 开头的文件（剔除第一行）
  • wc -l ：统计文件行数
    管道就是把前一个的输出作为后一个的输入继续操作。

一些类似 options：

  • ls -lR ：列出所有文件，包括子目录
  • grep ^d ：保留目录文件，筛去一般文件（- 开头）。

# 日志相关

  • 查找某一时间的日志： grep "2023-02-07 21:04:00" *.log
  • 查找日志最后 10 行： cat test.log | tail -n 10

# 输出重定向

例如 cat A.log > B.log

# 网络

# DNS 域名解析 / 输入一个网址之后发生什么

DNS：域名系统，域名 - IP

  1. 先查浏览器本地缓存，有没有本地缓存：有，返回 IP；没有，进入下一步（下同，省略）。
  2. 查找操作系统缓存。
  3. 把域名发送给本地 DNS 服务器，查找。
  4. 根 DNS 服务器请求解析，根服务器返回给本地 DNS 服务器第一个顶级 DNS 服务器地址。
  5. 本地 DNS 服务器向顶级 DNS 服务器请求解析。
  6. 顶级域名服务器找到 Name Server 域名服务器的地址（域名提供商的服务器），交由域名提供商解析。
  7. 域名提供商的服务器查询域名对应的 IP，返回 IP 和 TTL 给本地 DNS 服务器。
  8. 本地 DNS 服务器缓存 “域名 - IP” 对，把 IP 发还给浏览器。

# 主要解析记录类型

  1. A 记录：指向某个 IP
  2. CNAME 记录：为域名设置别名

# HTTP

# 特点

  1. 无连接：每次处理一个请求，响应之后即断开连接，无会话连接 —— 引入 Session 和 Cookie。
  2. 无状态：对之前的响应没有记忆，处理前置信息需要重传。

# GET 和 POST

  1. 都可以获取数据（可以用 POST 构造 GET），POST 有请求内容（body）。
  2. GET 可以在 url 中读出，POST 不行（写在报文内）。
  3. GET 有长度限制，POST 没有。

# 状态码

  1. 1XX：服务器示意请求者继续操作。
  2. 2XX：请求收到并成功处理。
  3. 3XX：重定向
  4. 4XX：客户端错误，请求包含语法错误 / 无法完成请求。
  5. 5XX：服务器错误，服务器在处理请求过程中发生错误。

301：资源被永久转移到其他 URL。
302：临时跳转。
400：Bad Request，请求中有语法错误。
404：请求资源不存在。
500：服务器发生不可预期错误。
503：Server Unavailable。

# HTTPS

针对以下缺陷修改：

  1. HTTP 的请求信息明文传输
  2. HTTP 没有完整性校验
  3. HTTP 没有身份校验

“S” means “over Secure Socket Layer”.
建立连接之后，服务器会返回 HTTPS 证书（一个公钥 PK\mathsf{PK}PK ）。
客户端生成对称密钥 K\mathsf{K}K ，用公钥加密 K\mathsf{K}K ，得到 Enc(PK,K)Enc(\mathsf{PK},\mathsf{K})Enc(PK,K) ，发还给服务器。
服务器用私钥 SK\mathsf{SK}SK 解密 Enc(PK,K)Enc(PK,K)Enc(PK,K) ，得到 K\mathsf{K}K 。
最后用对称密钥 K\mathsf{K}K 通信。

# Cookie

为了解决 HTTP 无状态的问题，在首次响应时，服务器生成 Cookie 一起发还给客户端。客户端下次建立连接时，凭借 Cookie 可以识别身份。
Cookie 容易被篡改。
最大大小为 4KB。

# Session

保存在服务器上。
在服务器上生成 id，将 id 作为 Cookie 返回给客户端。不将用户数据作为明文传输。

  2. 正确的账户名和密码
