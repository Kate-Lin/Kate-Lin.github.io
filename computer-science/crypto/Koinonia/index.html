# 报告简介

这篇课程报告主要是谈一谈阅读和学习 2019 年 ACSAC 会议 Koinonia: Verifiable E-Voting with Long-term Privacy 一文的收获和感悟。本学期选择这篇文章做汇报的原因是我本科的毕业设计也做过隐私保护的电子投票的工作，也接触过同态加密（Homomorphic Encryption, HE）和可验证秘密分享（Verifiable Secret Sharing, VSS）的知识，硕士阶段确定的方向是公钥加密，这样一篇文章可以说很好地连接了我在两个阶段的学习。

本篇报告将大致分以下几个方面展开：一，Koinonia 电子投票系统的论文概述；二，Koinonia 的新颖之处；三，在做完汇报后思考到的 Koinonia 论文的一些不足之处；四，我从这学期的论文学习和讲解过程中获得的收获。

# Koinonia 原文概述

Koinonia: Verifiable E-voting with Long-term Privacy 是发表在 ACSAC 2019 的 Privacy 板块下的一篇论文，Koinonia 在希腊语中有 “合作” 的意思，这也和方案核心的秘密分享相契合。整篇文章的贡献主要有三项：

  1. 分析了当前电子投票系统的未来攻击者模型分类，按攻击者的计算能力分为 current，future，unbounded 三类；按攻击者的信息观测能力分为 local，global 两类。
  2. 提出并论证了秘密共享比加密更适合保持长期安全的观点。
  3. 基于可验证秘密贡献构造了 Koinonia 电子投票协议，并有代码实现。

从文章的标题体现的两大要点就是 verifiable（可验证）和 long-term privacy（长期隐私）。可验证的目标是在投票的流程中，任意参与方都可以验证投票结果的正确性和无篡改，这主要是通过秘密分享，承诺和签名来实现的。长期隐私的目标是参与用户的隐私（譬如密码、投票选择等）不会被未来拥有无限算力和本地观测能力（unbounded & local）的攻击者破解，这主要是通过秘密分享来实现的。

综上，本篇论文是把 Pederson 承诺，零知识证明，数字签名，可验证秘密分享结合用于电子投票协议的方案，应用的技术点是比较综合的，并且有抗量子攻击的特性，这一点是比较值得被关注的。并且，该方案有代码实现公布在 GitHub 上，可以供验证测试和实际使用，目前的密码学研究的落地还不是那么广泛，我个人认为这也是使得论文更出彩的地方。

# Koinonia 的出彩之处

我认为这篇论文的出彩之处主要有以下几点：

  1. 对于研究的需求分析和调研非常详尽，目标明确；
  2. 对于安全性的考虑比较有前瞻性，不短视；
  3. 不盲目在加密上钻牛角尖，能够另辟蹊径；
  4. 作为应用项目，对于鲁棒性等各种特殊情况的考虑很周全。

第一点和第二点主要是根据 Koinonia 论文的第二章 An Framework of Adversaries 得出的。作者在原文中提到了，虽然电子投票协议和系统目前已被学界广泛研究，但是对于电子投票的攻击者的模型缺乏详尽的分析，并且由于缺乏详尽分析，方案缺乏长期的安全性，对抗量子攻击的电子投票方案的研究也不足。

许多电子投票协议仅可以抵抗当下的攻击能力的攻击者（即通常考虑的多项式时间内的敌手），常用公钥密码算法来保障安全性。这种级别的安全性被称为即时安全（immediate privacy）。但这样的隐私保护在不久的未来就可能会被打破。（随着量子计算的兴起，和新型密码算法的研究）所以作者在考虑实现长期安全（long-term privacy）。

作者就电子投票协议的长期安全给出了安全目标和威胁模型。长期安全应能抵抗来自未来的攻击。

所谓来自未来的攻击，指的是攻击者可能在未来对当前投票的隐私性进行攻击（如当下发布的密文信息，如选票记录等，可以在未来被还原）。作者敏锐地指出，当前对电子投票的研究有一些是偏离了重点方向的，如利用后量子密码来保障长期的密文完整性。因为从实际角度分析，纠结于当前投票的长期完整性是没有意义的，投票是当下的一个行为，计票和求和是在当下的时间就完成的，所以即使攻击者在未来有了可攻破经典公私钥密码的能力，也不会对选举的过程和结果产生任何的干扰。

因此，长期安全是要保障长期的隐私性而非长期的完整性。

这样的需求分析可以使研究者在制定方案时节省很多力气，毕竟不需要在每一个方面都应用高安全强度的密码方案。对于公钥密码，安全强度越高 / 越难攻破的方案，意味着更长的密钥和更复杂的困难问题，前者会导致运行时间变长，效率降低，后者比简单的困难问题更加难实现（比如基于 RLWE 的密码比基于 RSA 的密码更难以代码的形式实现）。对于对称密码也是类似的，更长的密钥意味着更长的运行时间。所以从应用角度而言，Koinonia 这样的出发点是很有利的，可以说是 “高效地做长远打算”。

关于 Koinonia 论文对需求分析和痛点的把控，还值得提到的就是调研并分析电子投票协议在底层信道是如何保障安全的。这也是在课上汇报和老师交流时，老师重点问到的一点。

我做毕业设计的时候也看过一些关于电子投票协议的文章（不过是更偏安全多方计算方向的），比如书籍《安全多方计算的研究与应用》和论文《基于安全多方计算的电子选举系统设计与实现》，其中也多次提到所谓 “安全信道”，但没有具体指出如何实现，这让我在写代码的时候非常迷惑，无从写起。所以在代码实现中，我就简单地把安全信道用 RSA 密码实现了。但是这样的信道是否真的是安全的？

Koinonia 的作者在文中指出，当前的研究往往是在假设诚实参与者在私密信道上传递信息，而私密信道目前往往是用计算安全（computationally secure）的密码方案进行构建的。但作者指出，这样的计算安全方案在未来会变得不安全，因为未来的敌手可能有更加强的计算能力。即使有抗量子攻击的公钥加密算法，当敌手的计算能力比量子攻击更强时，也会变得不安全。

为了理解敌手的攻击能力，作者提出了攻击者模型分类。按攻击者的计算能力分为 current，future，unbounded 三类；按攻击者的信息观测能力分为 local，global 两类，如图 1 所示。

  1. 具有多项式时间计算能力的 current 敌手，是经典公钥密码的防御对象；
  2. 具有量子攻击能力的 future 敌手，可攻破经典公钥方案，是后量子密码的防御对象；
  3. 具有无限计算能力的 unbounded 敌手可攻破后量子密码方案。
  4. 具有局部观测能力的 local 敌手，可获得半诚实方和恶意方的消息，和公开信息板上的消息；
  5. 具有全局观测能力的 global 敌手，不仅拥有上述 local 的消息，还可获得诚实方之间的通信内容。

对于长期隐私的实现，本文考虑防御的是 <local, unbounded> 敌手。原因有二：①图中顶端的 < global, unbounded > 敌手能力过强，由于有全局观测能力和无穷算力，即使使用无关计算能力的信息论安全的方案，也不能保证安全性；②位于图中同一高度的 < global, future > 方案，只需要对所有的消息应用后量子密码方案即可，可统一考虑。只有 < local, unbounded > 敌手的情况较为复杂，且攻击能力较强。

本文同样也提到了 “安全信道”，通过对本文上下文的理解，Koinonia 中使用的安全信道就是使用经典公钥密码的用于保密的信道，公钥加密，私钥解密，为什么这样的信道就可以保证面对 < local, unbounded > 敌手的长期安全呢？

其实作者在原文中没有给出特别完整的解释，原因比较碎片化，但是核心原因可以用一句话概括，那就是秘密分享的使用。秘密分享把明文信息分成若干个 share 之后，传送给多方分别保存，是一个和困难问题无关的方案，理论上具有信息论安全。秘密分享安全成立的条件是，电子投票协议中至少有一个诚实的计票人，协议就可以安全运行（后来我还想到了如果有诚实计票人，但是没有任何诚实的投票人的话，安全性也是荡然无存的，但是在电子投票中这样的情况太过极端，可能原文没有考虑进去）。

由于秘密分享的安全性与攻击者的计算能力并不关联，协议使用公钥加密安全信道传递 shares，即使存在非诚实方恶意公开安全信道中的密文，被无限计算能力的敌手破解明文，敌手可以拿到的也只有原文的分片。敌手的观测能力是局部的，且存在至少一个诚实方，所以敌手不能获得完整的明文，除非全部合谋。而且从某种意义上而言，即使直接用明文的分片传输，也不会影响安全性。

这样的安排不仅可以匹配长期安全的目标，并且也节省了安全的成本，用比较基础的加密方案实现了同样的效果，和上述提到的保障完整性一样，如果能达到目标，则少花额外的成本。

作者提出了使用秘密分享较加密的一些优势，是比较有建设性的，这也是我在本章开头的第三点提到的。在上文中，提到了计算安全的公钥方案在敌手计算能力不断增长的过程中会逐步失效。这样的 “防御 - 攻破 - 优化” 机制更类似于 “道高一尺，魔高一丈” 魔高一丈的打补丁防范。本文则直接跳出了先前的电子投票协议广泛采用加密的经验，转而考虑秘密分享，从一个新的角度构建安全方案。这并不代表加密的研究没有前途了，只是开辟了一条新的道路。

最后再讲一讲本文关于鲁棒性、选票完整性、选票合法性等系统运行特殊情况的考虑。这些考虑在论文中占了比较大的篇幅，作者总体考虑得比较详细。比如验证选票分片的完整性，用到了 Pederson 承诺，具有 unconditional hiding 和 computationally binding，可以防止进程中篡改和未来解密；验证选票格式正确（每个选民对每个候选人最多投一票）用到了见证不可区分证明（witness-indistinguishable proof）；在鲁棒性方面考虑了 DoS 攻击和恶意计票人的情况；在选票合法性方面还考虑了恶意选举机构的可能性。上述的考虑可以说是非常详尽了，这样的项目即使遭受了各种各样的恶意攻击，保持安全的可能性还是很大的。

# Koinonia 的不足之处

虽然上述讲到了 Koinonia 的很多出色之处，但是在阅读和讲解的过程中，我还是发现了这篇论文的一些不足之处。

最大的不足之处在于论文对于系统运行特殊情况的描述有一点堆砌，读到最后就有一些平淡。其实这些特殊情况的考虑，最重要就是承诺、零知识证明和签名，各种情况的处理其实都是应用这三项技术。但是在正文中，写得太繁杂了。验证完整性需要考虑计票人是否求和错误。鲁棒性考虑了三个方面：计票人不传计票和，计票人 DoS 攻击，计票人故意不对一些选票求和。选票合法性考虑了两个方面：选举服务提供者故意不放入选票，选举机构和选举服务提供者伪造选票，后者又细分四种可能性。

但是上述的情况区分，很多情况是重合的，一直在反复处理，讲得很累赘。总感觉有填充篇幅的感觉。在讲这篇论文的时候，讲到最后也感觉索然无味，一直在重复讲一些类似的情况。现在还记得老师和我说 “所以后面都是分析这些情况了是吗”，这一段确实是有点累赘了。

# 从 Koinonia 获得的收获和思考

这个学期的自选网络安全协议论文讲解大致有以下几点收获：

  1. 首先是反思和检验了一下自己在做本科毕业设计的时候用的思路和技术是否正确。本科毕业设计做的电子投票协议的实现是基于安全多方计算的，其中的很多思路都是和秘密分享共通的，比如将秘密分片传输给多个参与方。其实整体思路是很像的，但是自己当时根本也没有想这么多，只是想把前人的研究拿来做代码实现。并且我在实现协议的过程中，完全没有对恶意行为进行防范，即使当时不知道零知识证明的内容，但承诺和签名其实是完全可以实现的，但是我当时完全没有考虑。此外，做软件应用的需求分析我也不如作者做得好，即使这篇论文的实现可能不如 idea 看起来那么突破创新，但是想法和理念还是传达得很有延续性的。在以后的研究中也要多思考如何完善出一个不错的 idea。
  2. 在大四做毕业设计的时候，还考虑过把同态加密运用于电子投票协议，因为同态的性质允许密文的直接计算，解密得到明文结果。虽然对简单的应用而言也没有什么很大的安全威胁，但是看完这篇论文之后负责任地讲，这还是有一些隐患的。
  3. 重新回顾了电子投票协议的流程，参与方行为等知识。第一次接触 Pederson 承诺和见证不可区分证明（WIP），查阅了很多的相关资料，顺着每篇文章的参考文献不断地查，看了很多的证明，对前者了解比较深入了，但是对零知识证明的内容可能还是缺乏一些认识，即使到了讲论文的时候也没有理解得很好，虽然老师没有让我详细展开讲讲，但自己还是知道学得很不足的。虽然以后不是做零知识证明，但是应当了解的内容肯定不能只有定义而已。

总而言之，这个学期的收获还是很大的，不断学习新东西，结合已有的知识进行回顾，才可以走得更远，希望自己以后也可以保持这样学习新东西的动力。
