# 设置评论区头衔的问题

作者给的原始的 tagmeta 为 tagMeta:[visitor: 新朋友,master: 主人,friend: 小伙伴,investor: 金主粑粑] ，由于文章不设置打赏所以尝试把 invester 改成 author: 作者 ，但加入了自己的邮箱的 MD5 哈希后，始终无法识别 “作者” 的 title，一直显示新朋友。

  • solve: 可能是 author 标签和文章的 front matter 中的 author 标签重复？把 MD5 结果移到 master 栏目下，解决问题。

# 邮箱格式校验问题

原始评论区输入诸如 xxx.xxx@xxx.com 和 xxx_xxx@xxx.com 的邮箱都不能通过校验，初步判断是由于邮件校验正则有误。

  • 首先检查是否是_config.yml 的问题，发现没有和评论邮箱相关的设置。
  • 其次考虑在不修改原始的 MiniValine.min.js 文件的情况下，能不能在.js 文件中覆盖正则：
    发现还是不行，加了一个 click 的 function 之后直接卡死了。
  • 最后在 GitHub 上看到了有人提交了 bug，解决办法只有修改 MiniValine.min.js ，
    搜索 /^[A-Za-z0-9\u4e00-\u9fa5]+@[a-zA-Z0-9_-]+(\.[a-zA-Z0-9_-]+)+$/ ，修改为
    /^[A-Za-z0-9\u4e00-\u9fa5]+(\.[A-Za-z0-9\u4e00-\u9fa5]+){0,2}@[a-zA-Z0-9_-]+(\.[a-zA-Z0-9_-]+)+$/
  • 将新的 js 部署到 GitHub 仓库中，使用 https://cdn.jsdelivr.net/gh/XXX/YYY/ZZZ/MiniValine.min.js 为链接调用，在 vendors 中修改即为 valine: gh/XXX/YYY/ZZZ/MiniValine.min.js 。（XXX 为 GitHub 用户名，YYY 为 Github 仓库名，ZZZ 为 js 文件所在的文件夹）。

  • 在 vercel 官网先注册账号（可通过 GitHub 接入），然后在之前的命令行内输入 vercel login ，复制授权链接在浏览器内打开，完成登录。

  • 在 cdn.jsdelivr.net 文件夹内打开 cmd，输入部署命令 vercel -A cdn.jsdelivr.net.json --prod ，不需要 link to existing project，其余都可以选默认。

  • （可选）此后，进入 Vercel 官网，登录，进入应用控制台，点击刚才创建的项目，view Domains，Edit，Domain 出输入自定义的二级域名绑定应用，界面会弹出需要进行 DNS 解析的 CNAME，复制 CNAME 至 DNS 解析处（购买域名的地方有 DNS 解析），完成 DNS 解析。

  • 如果不绑定自定义的域名，也可以在 view domain 的地方直接复制 vercel.app 结尾的域名 DOMAINS （有概率被墙）

  • 重新部署项目和 MiniValine.min.js ，即可解决 cdn.jsdelivr.net 被墙的问题。
    解决的是评论区表情被墙的问题。

# 部署静态资源踩大雷：jsdelivr.net 的不定时刷新

  • 由于原版 MiniValine.min.js 的 GBUrl 和 EUrl 都是被墙的，所以需要修改后再次上传。
  • 但上传之后一直没有生效，查看调用的 MiniValine.min.js 发现仍然是修改之前的内容，搜索帖子发现 cdn.jsdelivr.net 的链接内容并不是实时刷新，往往一天左右才会更新。
  • Solve:
    • 第一种办法：将 MiniValine.min.js 存到项目文件夹的 theme/shoka/source/js/ 目录下，以 /js/ 相对路径或网站部署的绝对路径访问 js 文件，可参见 ReverseScale MiniValine 插件替换思路），这样可以保障每次部署都可以适应最新的 MiniValine 版本
    • 第二种办法：如果不想把 MiniValine.min.js 保存在项目里，仍然以 cdn.jsdelivr.net 调用 js 文件，需要把原仓库的 release 标明版本，以 https://cdn.jsdelivr.net/gh/XXX/YYY@Version/ZZZ/MiniValine.min.js 为链接调用，在 vendors 中修改即为 valine: gh/XXX/YYY@Version/ZZZ/MiniValine.min.js （Version 为 release tag）。

# 解决 mac 代码框无法正常显示的问题

很好笑，这个问题是在我写这篇文章的时候发现的，这是什么套娃行为，写 bug 经验测出 bug。

# 第一阶段 什么都显示不出

重新安装 hexo-renderer-multi-markdown-it ， hexo-renderer-ejs ， hexo-renderer-stylus 。

然后重新部署项目。

# 第二阶段 figcaption 不显示 data-lang

重装了三个渲染器之后虽然可以显示出 mac 的影子了，但是显示得完全不对。
看了源码之后发现 figcation 标签下没有 data-lang="xxx" 这项。
在 GitHub 的 Issue 上看到了完全一样的情况，但是提问者是没有设置 highlight: enable:false 这一项（ highlight 选项可以在 _config.yml 中搜索到），我之前已经设置了 false ，就很没有道理。
结果最后把这个 issue 的 config 又重新复制了一边，莫名其妙就行了，很没有道理。
如果有的时候没更新，也许是因为有缓存，非常相信自己的时候请 Ctrl + F5。
