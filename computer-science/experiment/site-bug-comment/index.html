# 设置评论区头衔的问题

作者给的原始的 tagmeta 为 tagMeta:[visitor: 新朋友,master: 主人,friend: 小伙伴,investor: 金主粑粑] ，由于文章不设置打赏所以尝试把 invester 改成 author: 作者 ，但加入了自己的邮箱的 MD5 哈希后，始终无法识别 “作者” 的 title，一直显示新朋友。

  • solve：可能是 author 标签和文章的 front matter 中的 author 标签重复？把 MD5 结果移到 master 栏目下，解决问题。

# 邮箱格式校验问题

原始评论区输入诸如 xxx.xxx@xxx.com 和 xxx_xxx@xxx.com 的邮箱都不能通过校验，初步判断是由于邮件校验正则有误。

  1. 首先检查是否是_config.yml 的问题，发现没有和评论邮箱相关的设置。

  2. 其次考虑在不修改原始的 MiniValine.min.js 文件的情况下，能不能在.js 文件中覆盖正则：
    发现还是不行，加了一个 click 的 function 之后直接卡死了。

    document.body.addEventListener('click', function(e)if (e.target.classList.contains('vsubmit')) {           const email = document.querySelector('input[type=email]');           const nick = document.querySelector('input[name=nick]');           const reg = /([A-Za-z0-9_\.\u4e00-\u9fa5]+@[a-zA-Z0-9_-]+(\.[a-zA-Z0-9_-]+)+$/;           if (!email.value || !nick.value || !reg.test (email.value)) {               const str = `请填写正确的昵称和邮箱！`;               const vmark = document.querySelector ('.vmark');               vmark.innerHTML = str;               vmark.style.display = 'block';               setTimeout (function () {                   vmark.style.display = 'none';                   vmark.innerHTML = '';), 2500);
    }
    }
    })

  3. 最后在 GitHub 上看到了有人提交了 bug，解决办法只有修改 MiniValine.min.js，
    搜索 /^[A-Za-z0-9\u4e00-\u9fa5]+@[a-zA-Z0-9_-]+(\.[a-zA-Z0-9_-]+)+$/ ，修改为
    /^[A-Za-z0-9\u4e00-\u9fa5]+(\.[A-Za-z0-9\u4e00-\u9fa5]+){0,2}@[a-zA-Z0-9_-]+(\.[a-zA-Z0-9_-]+)+$/

  4. 将新的 js 部署到 GitHub 仓库中，使用 https://cdn.jsdelivr.net/gh/XXX/YYY/ZZZ/MiniValine.min.js 为链接调用，在 vendors 中修改即为 valine: gh/XXX/YYY/ZZZ/MiniValine.min.js 。（XXX 为 GitHub 用户名，YYY 为 Github 仓库名，ZZZ 为 js 文件所在的文件夹）。
